Design Framework

Anti-Slop

Rejecting the average of all human designs. Enforcing intent, tension, and authorship. Not another generic design system.

// Core Philosophy

If an element can be moved to another product with zero loss of meaning, it is slop.

The Three Pillars

1. Asymmetry Over Balance

Balance is boring. Tension is memorable. Use Golden ratio splits (1.618:1), broken grids, and intentional misalignment to create visual friction.

grid-template-columns: 1.618fr 1fr;

2. Typography as Structure

If the type isn't doing the heavy lifting, the design is weak. Use violent scale jumps (4x-8x), negative letter-spacing, and dramatic hierarchy.

font-size: clamp(4rem, 15vw, 18rem);

3. Intentional Friction

We don't want "seamless." We want "meaningful." Overlapping elements, collisions, and visual speed bumps create engagement.

margin-top: -10%; /* Invade previous */

Material-Based Colors

Instead of pure values (#fff, #000), use materials with warmth and character:

--paper: #f9f9f7; /* Warm off-white */ --ink: #0d0d0b; /* Warm near-black */ --concrete: #e8e6e3; /* Mid-tone base */ --charcoal: #2a2826; /* Dark secondary */ --acid: #d4ff00; /* Spot accent (< 5%) */

These colors have names that describe their physical quality, not their HSL values.

Run the Audit

Before shipping any design, run the Slop Auditor to detect generic patterns.

// In browser console: SlopAuditor.check(); // For detailed report: SlopAuditor.generateReport();

The auditor checks for: generic border-radius, symmetrical layouts, weak typography, pure colors, generic icons, default gradients, and ambient animations.
