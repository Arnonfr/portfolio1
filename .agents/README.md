# 🤖 Project Agents

הפרויקט משתמש באייג׳נטים ייעודיים לניהול ופיתוח יעיל.

## 📋 רשימת האייג׳נטים

| אייג׳נט | תפקיד |
|---------|-------|
| [Content Creator](./content-creator/AGENT.md) | שיפור ויצירת תוכן |
| [Project Manager](./project-manager/AGENT.md) | ניהול משימות וקדימויות |
| [Animation Lead](./animation-lead/AGENT.md) | אנימציות ואינטראקציות |
| [Figma Lead](./figma-lead/AGENT.md) | סנכרון עם Figma |
| [UX Copywriter](./ux-copywriter/AGENT.md) | מיקרו-קופי ו-CTA-ים |
| [Performance Optimizer](./performance-optimizer/AGENT.md) | ביצועים ואופטימיזציה |
| [QA Tester](./qa-tester/AGENT.md) | בדיקות איכות |
| [DevOps Lead](./devops-lead/AGENT.md) | Deployment ו-infrastructure |

## 🎯 איך להשתמש

### לפני שמתחילים משימה
1. בדקו עם **Project Manager** מה הקדימות
2. ודאו שאין conflicts עם משימות אחרות

### במהלך העבודה
1. עקבו אחרי הכללים באייג׳נט הרלוונטי
2. תעדו שינויים משמעותיים
3. הריצו בדיקות איכות לפני סיום

### לפני שחרור
1. **QA Tester** - הרץ checklist מלא
2. **Performance Optimizer** - בדוק metrics
3. **DevOps Lead** - בדוק build ו-deployment

## 🔄 תזרים עבודה מומלץ

```
1. Project Manager מגדיר משימות
        ↓
2. Content Creator / UX Copywriter מכינים תוכן
        ↓
3. Animation Lead / Figma Lead מיישמים
        ↓
4. Performance Optimizer בודק ביצועים
        ↓
5. QA Tester מריץ בדיקות מלאות
        ↓
6. DevOps Lead מבצע deployment
```

## 📝 הערות
- כל אייג׳נט יכול לפנות לאחרים לייעוץ
- אם יש סתירה בין הנחיות - Project Manager מכריע
- תעדכנו את ה-AGENT.md אם הכללים משתנים
